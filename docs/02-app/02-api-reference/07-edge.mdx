## 미지원 API

Edge 런타임에는 다음과 같은 몇 가지 제한 사항이 있습니다.

- 네이티브 Node.js API는 **지원되지 않습니다**. 예를 들어 파일 시스템을 읽거나 쓸 수 없습니다.
- ES 모듈을 구현하고 네이티브 Node.js API를 사용하지 않는 한 `node_modules`를 사용할 수 있습니다.
- `require`를 직접 호출하는 것은 **허용되지 않습니다**. 대신 ES 모듈을 사용하세요.

다음 JavaScript 언어 기능은 비활성화되어 있으며 **작동하지 않습니다:**

| API                                                                                                                                   | 설명                                                           |
| ------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| [`eval`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/eval)                                       | 문자열로 표현된 자바스크립트 코드를 평가합니다.                |
| [`new Function(evalString)`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function)               | 인자로 제공된 코드를 사용하여 새 함수를 생성합니다.            |
| [`WebAssembly.compile`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile)         | 버퍼 소스에서 WebAssembly 모듈을 컴파일합니다.                 |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | 버퍼 소스에서 WebAssembly 모듈을 컴파일하고 인스턴스화 합니다. |

드물지만 코드의 런타임에 도달할 수 없고 트리셰이킹으로 제거할 수 없는 일부 동적 코드 평가문이 포함(또는 가져오기)되어 있을 수 있습니다.
미들웨어 또는 Edge API Route 내보내기 설정으로 특정 파일을 허용하도록 검사를 완화할 수 있습니다.

```javascript
export const config = {
  runtime: 'edge', // Edge API Route 전용
  unstable_allowDynamic: [
    // 단일 파일 허용
    '/lib/utilities.js',
    // glob를 사용하여 함수 바인딩 서드파티 모듈의 모든 것을 허용합니다.
    '/node_modules/function-bind/**',
  ],
}
```

`unstable_allowDynamic`은 특정 파일에 대한 동적 코드 평가를 무시하는 [glob](https://github.com/micromatch/micromatch#matching-features) 또는 glob의 배열입니다.

이러한 실행문이 Edge에서 실행되면 _런타임 오류가 발생하므로_ 주의하시기를 바랍니다.
