Next.js를 시작하는 가장 쉬운 방법은 `create-next-app`을 사용하는 것입니다. 이 CLI 도구는 모든 것이 설정된 새로운 Next.js 애플리케이션을 빠르게 구축할 수 있도록 해줍니다. Next.js의 기본 템플릿이나 [공식 Next.js 예시](https://github.com/vercel/next.js/tree/canary/examples) 중 하나를 사용하여 새로운 애플리케이션을 만들 수 있습니다. 시작하려면 다음 명령어를 사용하세요.

### 대화형

다음 명령을 실행하여 대화형으로 새 프로젝트를 생성할 수 있습니다.

```bash filename="Terminal"
npx create-next-app@latest

yarn create next-app

pnpm create next-app
```

이어서 다음과 같은 프롬프트가 나타납니다.

```bash filename="Terminal"
What is your project named?  my-app
Would you like to use TypeScript with this project?  No / Yes
Would you like to use ESLint with this project?  No / Yes
Would you like to use Tailwind CSS with this project?  No / Yes
Would you like to use `src/` directory with this project?  No / Yes
Use App Router (recommended)?  No / Yes
Would you like to customize the default import alias?  No / Yes
```

프롬프트에 응답을 마치고 나면, 응답에 기반하여 올바른 구성으로 새로운 프로젝트가 생성됩니다.

### 비대화형

명령행 인자를 전달하여 비대화형으로 새로운 프로젝트를 설정할 수 있습니다.

또한, 기본 옵션을 `--no-`라는 접두사를 붙여 무시할 수 있습니다.(예시, `--no-eslint`)

`create-next-app --help`를 참고하세요.

```bash filename="Terminal"
Usage: create-next-app <project-directory> [options]

Options:
  -V, --version                        버전 번호를 출력합니다.
  --ts, --typescript

    TypeScript 프로젝트로 초기화합니다. (기본값)

  --js, --javascript

    JavaScript 프로젝트로 초기화합니다.

  --tailwind

    Tailwind CSS 설정으로 초기화합니다. (기본값)

  --eslint

    ESLint 설정으로 초기화합니다.

  --app

    App Router 프로젝트로 초기화합니다.

  --src-dir

    `src/` 디렉터리 안에서 초기화합니다.

  --import-alias <alias-to-configure>

    import 구문에서 사용할 별칭을 지정합니다. (기본값 "@/*")

  --use-npm

    npm을 사용하여 애플리케이션을 구축하기 위해서 명령어에 명시합니다.

  --use-pnpm

    pnpm을 사용하여 애플리케이션을 구축하기 위해서 명령어에 명시합니다.

  --use-yarn

    Yarn을 사용하여 애플리케이션을 구축하기 위해서 명령어에 명시합니다.

  -e, --example [name]|[github-url]

    예시를 사용하여 애플리케이션을 구축합니다. 공식 Next.js 저장소의 예시 이름이나
    GitHub URL을 사용할 수 있습니다. URL은 모든 브랜치나 하위 디렉터리를 사용할
    수 있습니다.

  --example-path <path-to-example>

    가끔씩 GitHub URL이 슬래시가 포함된 브랜치 이름(예시, bug/fix-1)과 예시
    경로(예시, foo/bar)를 포함하고 있을 수 있습니다. 이때는 예시 경로를 개별적으로
    지정해야 합니다.
    --example-path foo/bar

  --reset-preferences

    저장된 기본 설정을 명시적으로 초기화합니다.

  -h, --help                           사용 정보를 출력합니다.
```

### Create Next App을 사용하는 이유는 무엇인가요?

`create-next-app`은 새 Next.js 애플리케이션을 순식간에 생성할 수 있습니다. 이는 Next.js 개발자들에 의해 공식적으로 유지되고 있고, 다음과 같은 많은 장점을 가지고 있습니다.

- **대화형 경험**: (인자 없이) `npx create-next-app@latest`를 실행하면 프로젝트 설정 과정을 안내하는 대화형 경험을 제공합니다.
- **의존성 없음**: 프로젝트 초기화가 매우 빠르게 완료됩니다. Create Next App은 의존성이 없습니다.
- **오프라인 지원**: Create Next App은 오프라인인 경우를 자동으로 감지하고 로컬 패키지 캐시를 사용하여 프로젝트를 구축합니다.
- **예시 지원**: Create Next App은 Next.js 예시 모음 중 한 가지를 사용하는 애플리케이션을 구축할 수 있습니다. (예시, `npx create-next-app --example api-routes`)
- **검증됨**: 해당 패키지는 Next.js 모노레포에 포함되어 있고 Next.js 자체와 동일한 통합 테스트 스위트를 사용하여, 모든 배포에서 예상한 대로 동작하도록 보장합니다.
